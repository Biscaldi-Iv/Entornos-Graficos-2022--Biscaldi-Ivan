Que es HTML5 y porque utilizarlo
HTML5 es la quinta revision mas importante del lenguaje de marcado HTML. Como novedad define criterios de conformidad para los navegadores.
Este lenguaje utilizado principalmente para definir la estructura de las paginas web, ahora tambien permite añadir audio y video sin necesidad de utilizar plugins. La reproduccion de estos pasa a ser responsabilidad del navegador. Esto hace que se pueda reducir el tamaño del archivo de la pagina.
Pueden añadirse videos subidos a paginas de terceros en la nueva pagina sin afectar elpeso del archivo.
Otra novedad es la geolocalizacion. Esto permite a los sitios detectar la ubicacion de los usuarios(la utilizacion de esta caracteristica debe ser notificada a los usuarios para no violar la privacidad de ellos).
Es posible hacer animaciones 2D mediante la etiqueta
Tambien pueden añadirse eventos de teclado, mouse u otro mando.
Una pagina que utiliza HTML5 es completamente accesible, se puede utilizar tanto en computadora como en dispositivos moviles.

Ventajas de HTML5
Es gratuito
Para crear documentos HTML no se necesita ningun programa especial, puede hacerse desde cualquier editor de texto. Al guardar el documento como HTML este sera visualizable desde cualquier navegador.
Codigo mas ordenado
La adicion de nuevas etiquetas ayudan a ubicar las partes de la estructura basica de toda pagina web. Es mas facil separar las etiquetas del contenido y asi los desarrolladores pueden trabajar de manera mas efectiva y detectar errores mas rapido.
Compatibilidad entre navegadores
Los navegadores modernos y mas populares soportan HTML5, lo que hace posible que casi todos los usuarios puedan visualizar el contenido sin problemas.
Almacenamiento mejorado
HTML5 cuenta con almacenamiento local. Esto permite evitar cookies y tener que enviar la informacion al servidor, lo que hace que la informacion se almacene de forma mas segura.
Diseño adaptativo
La compatibilidad de HTML5 con navegadores de dispositivos moviles hace que tambien puedan adaptarse las paginas para que sirvan tanto a ordenadores como dispositivos moviles. Esto puede hacerse con especificaciones en el documento o utilizando frameworks.
Formatos de audio
Los elementos de audio se indican con las etiquetas .
El comite de estandarizacion W3C deja a decision de los desarrolladores de navegadores que formatos de audio deben soportar los navegadores. Los formatos mas comunes son: .wav, .ogg, .au y .mp3

Dado que se desconoce que formatos de audio soporta el navegador de cada usuario es posible definir mas de un recurso para un elemento de audio. Para ello se utiliza la etiqueta por cada recurso que se proporcione. El navegador reproducira el primer recurso que soporte.
