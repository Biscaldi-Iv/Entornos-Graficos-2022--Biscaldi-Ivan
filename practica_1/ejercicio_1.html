Cuestionario

¿Qué es HTML, cuando fue creado, cuáles fueron las distintas versiones y cuál es la última?

HTML(HiperText Markup Language) es un lenguaje de marcado con hipertexto adoptado como lenguaje de publicacion de la World Wide Web Consortium (W3C). HTML es un estandar que define una estructura basica y codigo(codigo HTML) para la definicion de contenido de una pagina web. W3C es una organizacion dedicada a la estandarizacion de tecnologias que se relacionen con la web. HTML ha tenido diferentes versiones:

  1. HTML 2.0: En 1995 se publica el estándar HTML 2.0. A pesar de su nombre, HTML 2.0 es el primer estándar oficial de HTML. No soportaba tablas. Se simplificaba al máximo la estructura del documento para agilizar su edición, la declaración explícita de los elementos body, html y head era opcional.
  2. HTML 3.2:se publicó en 1997 y es la primera recomendación de HTML publicada por el W3C. Incorporó los últimos avances de las páginas web desarrolladas hasta 1996, como applets de Java y texto que fluye alrededor de las imágenes
  3. HTML 4.01: se publicó en diciembre de 1999 e iba a ser la ultima publicacion oficial de HTML. Desde la publicación de HTML 4.01, el W3C se centró en el desarrollo del estándar XHTML. Por este motivo, en el año 2004, las empresas Apple, Mozilla y Opera mostraron su preocupación por la falta de interés del W3C en HTML y decidieron organizarse en una nueva asociación llamada WHATWG (Web Hypertext Application Technology Working Group) que comenzó el desarrollo del HTML 5
  4. HTML 5, 5.1 y 5.2: el WHATWG publico borradores de HTML 5.0 en marzo de 2007 y esto impulso a W3C a continuar estandarizando HTML, de hecho decidio integrar XHTML. Tras una larga evolucion, en octubre de 2014 W3C libera el estandar HTML 5, el cual incorporaba nuevos elementos, etiquetas y la posibilidad de introducir audio y video sin necesidad de plugins o complementos de navegadores
HTML 5 es la revision mas reciente, entre las mas importantes, del lenguaje. Ha definido criterios de conformidad para todos los agentes de usuarios(navegadores) e intenta mejorar la habilidad de diferentes sistemas para intercambiar informacion.

¿Cuáles son los principios básicos que el W3C recomienda seguir para la creación de documentos con HTML?

Separar estructura y presentacion: Dado que a medida que evoluciona HTML pueden verse reemplazados sus elementos y atributos de presentacion por otros, resulta conveniente que la estructura sea independiente de la presentacion. Ademas esto permite que una misma pagina sirva para diferentes plataformas.
Considerar la accesibilidad universal: los autores deberia considerar como representar sus documentos en diferentes plataformas(navegadores basados en voz, lectores braille, etc) para que la web pueda ser mas accesible para todos (como por ejemplo personas con discapacidades). Ademas es posible que algunos sitios cuenten con audiencias muy lejanas que tengan computadoras y configuraciones muy diferentes, y para ellos seria conveniente que los documentos puedan internacionalizarse.
Ayudar con la representacion incremental: usando tablas y/o otras nuevas caracteristicas incrementales es posible ayudar a los navegadores a representar los documentos mas rapido.

En las Especificaciones de HTML, ¿cuándo un elemento o atributo se considera desaprobado? ¿y obsoleto?

Se considera desaprobado a todo elemento o atributo que resulte anticuado por la existencia de nuevas estructuras. Estos deberian continuar teniendo soporte por parte de los navegadores pero en un futuro dejaran de tenerla.
Se considera obsoleto a todo elemento o atributo que ya no tiene garantia de soporte por parte de un navegador. Un elemento o atributo obsoleto a la vez, es aquel que ya no es definido especificaciones de HTML(a excepcion de que se lo nombre por cuestiones historicas).

¿Qué es el DTD y cuáles son los posibles DTDs contemplados en la especificación de HTML 4.01?

El DTD es la declaración del tipo de documento(document type declaration). Este indica que version de HTML utiliza el documento.
Los DTD especificados por HTML 4.01 son:

  • Strict DTD(DTD estricto): incluye a todos los atributos y elementos que no fueron desaprobados o que no aparecen en documentos con marcos.
  • Transitional DTD(DTD transicional): similar al DTD estricto pero tambien incluye elementos y atributos desaprobados.
  • Frameset DTD(DTD con marcos): incluye los elementos y atributos del DTD transicional y tambien los marcos.

¿Qué son los metadatos y cómo se especifican en HTML?

Los metadatos son informacion acerca del documento. Indican sus propiedades(autor, fecha de caducidad, palabras claves, etc). Entre otros permite establecer el idioma, de forma que los sintetizadores de voz puedan usar reglas de pronunciacion que son dependientes del idioma.
